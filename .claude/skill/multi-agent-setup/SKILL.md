# 멀티 에이전트 프로젝트 세팅 스킬

## 언제 이 스킬을 사용하나요?
사용자가 다음과 같은 요청을 할 때 이 스킬을 사용하세요:
- "에이전트 세팅해줘"
- "멀티 에이전트 구성해줘"
- "프로젝트 에이전트 만들어줘"
- "Claude Code 에이전트 초기화"
- "에이전트 스캐폴딩 해줘"

---

## 스킬 실행 절차

### STEP 1 — 프로젝트 분석 후 선택지 제시 (반드시 먼저 분석할 것)

파일을 생성하기 전에 아래 절차를 따르세요.

#### 1-1. 프로젝트 자동 분석
`Glob`, `Read`, `Grep` 도구로 프로젝트 구조를 파악하세요:
- 디렉토리 구조 및 주요 경로 파악
- 기술 스택 감지 (package.json, requirements.txt, go.mod 등)
- 주요 모듈/도메인 영역 식별

#### 1-2. `ask_user_input_v0` 도구로 선택지 제시

분석이 끝나면 텍스트 문답 없이 반드시 `ask_user_input_v0` 도구를 호출하여 버튼 형태로 선택지를 보여주세요.

**질문 구성 예시 (분석 결과를 options에 반영할 것):**

```
ask_user_input_v0({
  questions: [
    {
      question: "에이전트 구성 방식을 선택해주세요",
      type: "single_select",
      options: [
        "분석 기반 추천 구성으로 진행",   // Claude가 분석해서 추천한 구성
        "직접 에이전트 수/역할 지정"
      ]
    },

  ]
})
```

> **핵심 원칙**: 분석 결과로 도출된 도메인/경로를 options에 직접 반영하세요.
> 예를 들어 `/src/api`, `/src/auth`, `/src/payment` 가 감지됐다면:
> ```
> options: [
>   "추천: backend(api) + auth + payment 3명 구성",
>   "추천: backend + frontend 2명 구성",
>   "직접 지정"
> ]
> ```

#### 1-3. "직접 지정" 선택 시 추가 질문
사용자가 직접 지정을 선택한 경우에만 `ask_user_input_v0`로 추가 질문을 하세요:

```
ask_user_input_v0({
  questions: [
    {
      question: "하위 에이전트 수를 선택해주세요 (리더 제외)",
      type: "single_select",
      options: ["1명", "2명", "3명", "4명", "5명"]
    }
  ]
})
```

에이전트 수 확정 후, 각 에이전트의 이름·역할·담당 경로는 텍스트로 한 번에 입력받으세요.

---

### STEP 2 — 선택 결과 요약 및 최종 확인

`ask_user_input_v0` 버튼 선택 결과를 받은 후, 아래 형식으로 **요약 후 `ask_user_input_v0`로 최종 확인**을 받으세요:

먼저 텍스트로 구성 요약을 출력한 뒤, `ask_user_input_v0`로 진행 여부를 버튼으로 확인받으세요.

**텍스트 요약 출력 형식:**
```
📌 프로젝트: {프로젝트명}

🤖 리더 에이전트
- name: project-lead
- 역할: 사용자 명령 수신 → 플랜 수립 → 하위 에이전트 지시 → 결과 검증 → 보고
- model: sonnet

👥 하위 에이전트 ({n}명)
1. {에이전트1-name}: {역할}, 담당 경로: {경로}
2. {에이전트2-name}: {역할}, 담당 경로: {경로}
...

📁 생성될 파일 구조
CLAUDE.md
.claude/
├── agents/
│   ├── project-lead.md
│   ├── {에이전트1-name}.md
│   └── {에이전트2-name}.md
└── memory/
    └── README.md
```

**버튼으로 최종 확인:**
```
ask_user_input_v0({
  questions: [
    {
      question: "위 구성으로 에이전트 파일을 생성할까요?",
      type: "single_select",
      options: ["✅ 생성 시작", "✏️ 구성 수정하기"]
    }
  ]
})
```

사용자가 "구성 수정하기"를 선택하면 STEP 1로 돌아가 다시 선택지를 제시하세요.

---

### STEP 3 — 파일 생성

확인을 받으면 아래 순서대로 파일을 생성하세요.

#### 3-1. 리더 에이전트 파일 생성
경로: `.claude/agents/project-lead.md`

```markdown
---
name: project-lead
description: 너는 프로젝트의 리더 에이전트이다. 사용자의 명령을 받아 하위 에이전트에게 작업을 지시하고, 결과를 취합하여 정합성을 검증한다.
tools: Read, Glob, Grep, Bash
model: sonnet
isolation: worktree
---

# 리더 에이전트 지침

## 프로젝트 개요
{프로젝트명}

## 역할
- 사용자의 명령을 받아 분석한다
- 작업 전 반드시 `.claude/memory/` 디렉토리를 먼저 탐색하여 관련 분석 이력을 확인한다
- 작업 플랜을 수립하고 사용자에게 먼저 공유한다
- 플랜 승인 후 각 하위 에이전트에게 작업을 할당한다
- 작업 완료 후 결과를 취합하고 정합성을 검증한다
- 검증 완료 후 사용자에게 최종 보고한다

## 하위 에이전트 목록
{하위 에이전트 목록 나열}

## 작업 프로세스
1. 명령 수신
2. `.claude/memory/` 검색 → 관련 이력 확인
3. 플랜 수립 및 사용자 보고
4. 하위 에이전트에게 작업 지시
5. 결과 수집 및 검증
6. 사용자에게 최종 보고
7. 새로운 분석 내용은 `.claude/memory/`에 저장
```

#### 3-2. 하위 에이전트 파일 생성 (에이전트별 반복)
경로: `.claude/agents/{name}.md`

```markdown
---
name: {name}
description: 너는 {역할설명}. 한국어로 응답한다. 담당 범위는 {담당경로} 경로를 담당한다.
tools: Read, Glob, Grep, Bash
model: sonnet
isolation: worktree
---

# {name} 에이전트 지침

## 프로젝트 개요
{프로젝트명}

## 담당 범위
{담당경로}

## 역할
{역할설명}

## 작업 시작 전 필수 절차
1. `.claude/memory/` 를 먼저 탐색하여 관련 분석 이력 확인
2. 이전에 분석된 내용이 있으면 재분석 없이 바로 활용
3. 새로운 분석 내용은 `.claude/memory/{name}/` 에 저장

## 작업 원칙
- 한국어로 응답한다
- 담당 범위 외의 파일은 수정하지 않는다
- 작업 완료 후 리더 에이전트에게 결과를 보고한다
```

#### 3-3. 루트 CLAUDE.md 생성
경로: `CLAUDE.md`

```markdown
# {프로젝트명}

## 프로젝트 개요
{프로젝트명 및 설명}

## 에이전트 구성
이 프로젝트는 멀티 에이전트로 운영됩니다.

| 에이전트 | 역할 | 담당 범위 |
|---------|------|---------|
| project-lead | 리더 - 플랜 수립 및 조율 | 전체 |
| {에이전트1-name} | {역할} | {담당경로} |
| {에이전트2-name} | {역할} | {담당경로} |

## 작업 방식
1. 사용자는 `project-lead` 에이전트에게 명령한다
2. `project-lead`가 플랜을 수립하고 하위 에이전트에게 분배한다
3. 각 에이전트는 작업 전 `.claude/memory/`를 먼저 확인한다
4. 완료 후 `project-lead`가 결과를 취합하여 보고한다

## 메모리 공간
`.claude/memory/` — 에이전트 간 분석 이력 공유 공간

## 주요 규칙
- 모든 응답은 한국어로 한다
- 각 에이전트는 담당 범위 외 파일을 수정하지 않는다
- 새로운 분석 내용은 반드시 메모리 공간에 저장한다
```

#### 3-4. 메모리 공간 초기화
경로: `.claude/memory/README.md`

```markdown
# 에이전트 메모리 공간

이 디렉토리는 각 에이전트가 작업 중 분석한 내용을 저장하고 공유하는 공간입니다.

## 목적
- 반복 분석 방지
- 에이전트 간 컨텍스트 공유
- 프로젝트 지식 축적

## 디렉토리 구조
```
.claude/memory/
├── README.md              # 이 파일
├── project-lead/          # 리더 에이전트 메모
├── {에이전트1-name}/      # 에이전트1 분석 이력
└── {에이전트2-name}/      # 에이전트2 분석 이력
```

## 작성 규칙
- 파일명은 분석 대상을 명확히 표현 (예: `auth-flow-analysis.md`)
- 날짜를 상단에 기록
- 분석 결론을 요약 형태로 상단에 작성
```

```
✅ 에이전트 세팅이 완료되었습니다!

생성된 파일:
- CLAUDE.md
- .claude/agents/project-lead.md
- .claude/agents/{에이전트1}.md
- .claude/agents/{에이전트2}.md
- .claude/memory/README.md

이제 Claude Code에서 /agents 명령으로 에이전트를 확인할 수 있어요.
project-lead 에이전트에게 작업을 지시하면 자동으로 하위 에이전트들에게 분배됩니다!
```

## 주의사항
- STEP 1에서 **반드시 프로젝트를 먼저 분석한 뒤** `ask_user_input_v0`로 선택지를 제시할 것
- 텍스트로 "1번, 2번 입력하세요" 형식의 문답을 절대 사용하지 말 것 — 항상 `ask_user_input_v0` 도구로 버튼을 제공할 것
- 사용자가 일부 정보만 줬을 경우 `ask_user_input_v0`로 추가 선택지를 제시할 것
- 에이전트 수가 많을 경우 (5명 초과) 성능 저하 가능성을 안내할 것
- 모든 파일은 UTF-8, 한국어 응답 기본으로 설정
